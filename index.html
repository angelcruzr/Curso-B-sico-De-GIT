Tutorial De Git, Nivel Básico

Aprende todos los comandos y acciones indispensables para trabajar con el mejor y, por lo menos, más usado Sistema de Control de Versiones

Las aplicaciones que usaremos durante este tutorial son la terminal de comandos Git Bash y el editor de código Visual Studio Code. Con paciencia y constancia comprobarás tus progresos en el conocimiento y manejo de Git.

Espero que te lo pases bien aprendiendo esta herramienta tan importante y usada por la mayoría de profesionales del mundo de la programación y también por aprendices autodidactas, como yo mismo.

Parte 1
Comandos Imprescindibles Para Comenzar

Primero, abre Git Bash. Si la instalación de Git se ha completado satisfactoriamente, busca abajo y a la izquierda de tu ordenador y situa el mouse encima del icono de Windows ( si tienes su sistema operativo instalado en tu pc).

Busca la carpeta "Git". Para tenerlo a mano tienes la opción de anclarlo en la barra de tareas.

Configurar Datos Personales: Nombre y E-mail

Sin un nombre de usuario y sin una dirección de correo electrónico, Git no nos dejará inicializar repositorios para nuestros proyectos.

Configurar Nombre De Usuario

Aquí te muestro cómo escribir el comando para crear el nombre de usuario. El símbolo $ ya está escrito por defecto en la terminal de Git Bash.

Para escribir texto con espacios, para crear carpetas y nombre de usuario, es necesario envolverlo entre comillas para no provocar un error.

Copia los códigos que necesites y deja solo un '$' a la izquierda: 

                        
                            $ git config --global user.name "Nombre Usuario"

Con git config --global especificamos que los proyectos que tengamos guardados en nuestro ordenador o Pc estarán accesibles para que cualquier usuario pueda trabajar con Git desde dicho dispositivo.

Configurar E-mail

El proceso para la configuración del correo e.co será parecido al paso anterior.

Para escribir símbolos como la @:
1º- Click derecho del ratón sobre la propia terminal => 2º- Options... => 3º- Keys => 4º- Marca la casilla 'Crtl+LeftAlt is AltGr'. Por último, haz click en 'Apply' y después en 'Save'. 

                        
                            $ git config --global user.email "miemail@.123"

Sustituye esta dirección de correo electrónico por la que quieras. Cuando abras una cuenta en Github deberás proporcionar una dirección de email para autentificar tus datos y para que puedas realizar tus trabajos con normalidad.

Tras estos pasos, podrás comprobar que se ha creado un archivo .gitconfig en la siguiente ruta: C:\Users\Nombre con los datos que has introducido. (Nombre: El que tengas establecido).

Parte 2
Crear Carpetas Para Repositorios

Para no hacer algo complicado el trabajo con Git Bash al principio, vamos a crear una carpeta en el escritorio del dispositivo.

Acceder A La Ruta Del Escritorio

Recuerda: El símbolo '$' (Dólar) aparecerá por defecto en los pasos que vayamos efectuando en la terminal. Si quieres, copia los ejemplos de código que necesites.

Copia solo las líneas con el texto en color blanco.

Escribe cd en la terminal y haz click. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~
                            $ cd

De esta manera te situarás en la raiz del directorio, en el punto de inicio desde el cual podrás acceder a cualquier carpeta o archivo que se encuentre en tu dispositivo.

A partir de aquí iremos utilizando comandos muy sencillos de ejecutar y fáciles de recordar.

Crear Carpetas

Cuando tengas soltura con la terminal, podrás crear repositorios donde estimes oportuno. Ahora vamos a lo más sencillo. Para ello escribe por primera vez ls y haz click. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~
                            $ ls

Cuando hayas hecho el correspondiente click, aparecerán en pantalla todas las carpetas, archivos y rutas existentes en tu ordenador.

Ahora nos interesa la ruta hacia el escritorio. Ésta será Desktop.

cd significa "Change Directory", o sea, cambiar de directorio.

Siempre podrás usar ls para saber qué elementos hay en cada ruta que vayas recorriendo. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~
                            $ cd Desktop/

Nos encontramos en el escritorio (~/Desktop). Veamos cómo crear una carpeta: 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop
                            $ mkdir "Nombre Carpeta"

mkdir es la abreviatura de "Make Directory" = Hacer / Construir Directorio.

Haz click y comprueba que la carpeta creada con el nombre que introdujiste se encuentra en el escritorio de tu dispositivo.

Las comillas nos permiten ecribir nombres de referencia, como los de las carpetas, con espacios de separación entre palabras. (También puedes introducir números). También se pueden poner guiones entre palabras; Por buenas prácticas, lo más recomendable es que usemos siempre las comillas.

Cómo Eliminar Carpetas

Si te has equivocado escribiendo el nombre que quieres darle a la carpeta o has cometido algún error ortográfico, no te preocupes. Puedes eliminarla con rmdir, que quiere decir 'remove directory'; En español podríamos decir 'remover directorio', para retenerlo mejor en nuestra memoria. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop
                            $ rmdir "Nombre Carpeta"

Vuelve a crear la carpeta como hiciste anteriormente.

Comandos Que Servirán Como Orientación

Ya hemos visto algunos comandos que nos han servido para acceder a determinadas rutas y saber qué encontrábamos dentro de ellas.

Retroceder Hacia La Ruta Anterior

Con el siguiente comando podremos retroceder un paso. Nos servirá para movernos con mayor soltura por la terminal o por si nos hemos equivocado de ruta: (deja un espacio entre 'cd' y los dos puntos). 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop
                            $ cd ..

Reconocer La Ruta Actual

Para saber dónde estamos situados y estar seguros de que vamos a trabajar en la carpeta o archivo deseado: 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop
                            $ pwd

En este caso puede resultar demasiado obvio, pues no hemos accedido a rutas como carpetas que se encuentran dentro de otras carpetas y que, a su vez, contienen varios archivos. etc.

El símbolo ~ (virgulilla), sirve como "punto de referencia" de los movimientos que vamos realizando por medio de Git Bash. A su derecha irán apareciendo los elementos que componen la ruta que estamos recorriendo.

Limpiar La Terminal

Si necesitas ver con mayor comodidad lo que va apareciendo en la pantalla, puedes limpiarla escribiendo clear y dando click. No perderás nada del trabajo que hayas realizado hasta el momento.

Arriba, en la terminal, tendrás disponible una línea de código para saber dónde estabas trabajando antes de limpiar la pantalla. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop
                            $ clear

Volver A La Raiz De Contenidos Del Dispositivo

Ya hemos visto este comando al principio de este tutorial, pero es bueno recordarlo para empezar de nuevo y realizar las pruebas necesarias para tener bajo control todo lo aprendido hasta ahora:

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop
                            $ cd

Parte 3
Interacción Entre Git Bash Y Visual Studio Code

Hasta ahora hemos trabajado solo con la terminal de Git Bash. Para que veamos la progresión de nuestro proyecto trabajaremos en paralelo con Visual Studio Code.

Primeros Comandos

Ya sabemos que debemos utilizar cd para cambiar de directorio y, a continuación, la ruta de la carpeta que nos interesa. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop
                            $ cd "Nombre Carpeta"

Haz click para tener dicha carpeta como ruta principal para poder crear archivos y carpetas. Luego escribe ls. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta
                            $ ls

Al hacer click verás que la carpeta está vacía.

Abrir El Proyecto En Visual Studio Code Desde Git Bash

Empecemos con la parte más interesante y entretenida de este tutorial. Para ello solo tienes que escribir code . y hacer click. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta
                            $ code .

Pero, ¿Cómo vamos a "inicializar" el repositorio?. No hay ningún archivo ni carpetas aún.

Inicializar Repositorio

He puesto Inicializar entre comillas porque es un verbo muy utilizado en el mundo de la programación y que, al principio, puede resultar un poco intimidatorio.

Es algo tan sencillo como escribir git init, sin perder nunca de vista que estamos en la raiz del proyecto. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta
                            $ git init

Veamos primero qué aparece debajo del propio comando que acabamos de ejecutar: 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta
                            $ git init
                            Initialized empty Git repository in C:/Users/Nombre/Desktop/Nombre Carpeta

Se puede traducir como "inicializado el repositorio vacío de Git en...", seguido por la ruta completa.

En segundo lugar, comprobaremos cómo se denomina la rama principal de todo este repositorio: (master).

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta (master)
                            $

La palabra master en inglés tiene un sentido, entre otros, de "dueño" o "amo", por lo cual la mayoría de usuarios de Git y Github optamos por el término main, en español "principal".

Cómo Cambiar El Nombre De La Rama Principal

Por los motivos ya citados, y si te parece bien, vamos a cambiar el nombre de "master" a "main".

Para ello escribimos el comando git branch -m main 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta (master)
                            $ git branch -m main

Dando click tendremos la rama principal con el nombre main.

Crear Primer Archivo Desde Git Bash

A partir del siguiente comando podremos ver los primeros cambios que se producirán en Visual Studio Code.

Con touch seguido por el tipo de archivo con el que quieras trabajar.

En mi caso voy a crear un archivo .html: touch index.html. En este caso, el nombre del archivo completo no va entre comillas. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta (main)
                            $ touch index.html

En la siguiente imagen comprobaremos qué tenemos en el editor de código.

U significa "No Rastreado". En el siguiente apartado veremos cómo completar cada tarea.

Parte 4
Comandos Para El Control De La Rama 'main'

Este tutorial está dedicado al aprendizaje de Git usando Git Bash. Por tal razón, vamos a prescindir de Visual Studio Code por el momento.

Cómo Conocer El Estado De La Rama (main)

Para no provocar el caos en nuestro proyecto, tenemos varios comandos que nos permiten controlar el trabajo hasta ahora realizado.

Editar Tareas Incompletas

Con git status sabremos si vamos haciendo correctamente nuestro trabajo. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta (main)
                            $ git status

Recuerda que habíamos creado el archivo index.html, el cual aparecía en el editor de código como archivo pendiente de ser 'reconocido'.

Observa cómo nos advierte la terminal de Git Bash de que dicho archivo aún no está rastreado. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta (main)
                            $ git status
                            On branch main
                            No commits yet
                            Untracked files:
                              (use "git add <file>..." to include in what will be committed)
                                   index.html
                            nothing added to commit but untracked files present (use "git add" to track)

Veamos de qué nos advierte este mensaje: En primer lugar, en la rama 'main' no hay todavía ningún commit; después nos informa de los archivos que no han sido rastreados. Vemos en rojo el archivo al que se refiere. Si existieran más acciones por terminar saldrían reflejadas de la misma manera.

Lo vamos a solucionar en el siguiente punto.

git add

Al final del anterior mensaje en la consola de Git Bash leemos (use "git add" to track).

Ahora debemos utilizar git add index.html para rastrear o reconocer el archivo anteriormente creado o modificado. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta (main)
                            $ git add index.html

Comandos Para Finalizar Tareas

¿Recuerdas que dentro del mensaje tras el git status nos refería No commits yet?.

Aún resta hacer un commit para poder realizar la primera “fotografía”. (Commit = Ingresar, internar, remitir).

git commit

Este comando nos servirá para crear un mensaje que nos permita identificar y recordar el porqué de la existencia del archivo creado.

Para evitar cualquier tipo de confusión, nos vamos a limitar a usar el mínimo número de comandos. Hay varios conceptos a tener en cuenta en el futuro en los que podrás profundizar, una vez tengas controlado este tutorial básico.

El comando a usar es git commit -m "mensaje". 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta (main)
                            $ git commit -m "Mensaje que quieras"

Una vez ejecutado, obtendremos la siguiente información: 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta (main)
                            $ git commit -m "Mensaje que quieras"
                            [main (root-commit) ef53915] Mensaje que quieras
                             1 file changed, 0 insertions(+), 0 deletions(-)
                             create mode 100644 index.html

Comprobar Éxito Final

¿Cómo podemos saber si todo ha salido bien?. Lo habrás adivinado: Con otro git status. 

                        
                            Nombre@LAPTOP-KLFF86 MINGW64 ~/Desktop/Nombre Carpeta (main)
                            $ git status
                            on branch main
                            nothing to commit, working tree clean

on branch main, es decir, en la rama principal de nuestro proyecto, nothing to commit, working tree clean, nada quedará sin su correspondiente commit y que estamos trabajando con el "árbol" limpio.
